Context¶

The contexts defines list of elements selectable to define ticket context.

Contexts are initially set to be able to define contexts for IT Projects, for three context types :

  • Environment
  • Operating System
  • Browser

They can be changed to be adapted to any kind of project.

Section: Description

Field Description
Id Unique Id for the context.
Context type One on the three context type.
Name Name of the context.
Sort order Number to define order of display in lists
Closed Flag to indicate that context is archived.

* Required field

Fields: Context type

  • List is fixed.
  • Captions are translated and so can be changed in language file.
