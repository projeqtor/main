Concernant les éléments d'avancement, voici les éléments qui vous permettront d'avancer :
  Sur chaque élément "planifiable" (projet, activité), les éléments : 
     Date de début, 
     Date de fin, 
     Durée,
     Charge 
sont présentés sur plusieurs niveaux :
     Demandé, 
     Validé, 
     Assigné (uniquement pour les charges), 
     Planifié, 
     Réel, 
     Reste (uniquement pour les charges).

Les niveaux "Demandé" et "Validé" sont directement saisissables et correspondent aux engagements (demandés et acceptés).
La charge assignée est directement calculée à partir des assignations sur les activités (somme des charges).
Les données planifiées sont calculées par le processus de Planification à partir des charges (menu Suivi / Planning) et non modifiables.
Les données réelles sont calculées à partir des données d'imputation (charges) et du statut de l'élément.
La charge restante est directement restituée à partir des données saisies sur l'écran des imputations.

Donc les colonnes "Planifié", "Réel" et "Reste" sont non modifiables.
Si ce comportement vous dérange, vous pouvez rendre ces données modifiables en modifiant le code :
  Sur le fichier, \model\PlanningElement.php, retrouver la définition :
     private static $_fieldsAttributes=array(
  et enlever le statut "readonly" sur le(s) champ(s) concerné(s).
  Au besoin, faire de même sur les fichiers \model\ProjectPlanningElement.php et \model\ActivityPlanningElement.php qui peuvent redéfinir ces propriétés.
